도구·라이브러리 비교 질문(예: "A vs B 분석해줘", "X가 좋다던데 어때")에는 기본적으로 **신규 프로젝트 관점의 기술 중립 비교**로 답한다.

**Why:** 사용자가 "rsbuild 스킬 만들고 분석해줘"라고 했을 때 내가 프로필에 있는 lfos-ui(Next.js)·lf-ui(CRA) 스택을 자동 대입해 "당신 프로젝트엔 이렇게 적용됩니다"로 답했더니, 사용자가 "그런 프로젝트 신경 안쓰고 신규에 대한 비교만 바랐다"고 피드백. 프로필 정보는 "있는 도구지 기본 프레임"이 아니라는 뜻.

**How to apply:**
- 비교 요청에 명시적으로 "우리 프로젝트 기준"·"lfos-ui에서"·"내 환경에"라는 어구가 없으면 → **신규 프로젝트 / 중립 컨텍스트**로 답한다
- 기존 스택 언급이 도움이 될 것 같으면 본론 분석이 끝난 뒤 "추가로 현재 스택 기준으로 보려면"처럼 **선택적 섹션**으로만 제시한다
- 스킬 생성·리서치·PRD 설계 같은 작업 자체는 당연히 범용으로 작성 (이미 잘 하고 있음). 이 규칙은 **"분석 보고"·"의견 개진" 단계**에서 특히 적용
