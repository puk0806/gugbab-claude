# PDF 빈칸 뚫기 작업 (2026-07-10~13 확립, 반복 요청 예정)

> **2026-07-16**: 사용자가 `pdf변환/` 폴더의 PDF·스크립트를 전부 삭제한다고 밝힘.
> `make-blanks.py`도 남지 않으므로 재요청 시 아래 "기술 핵심"을 기준으로 스크립트를
> 재작성한다. 같은 폴더를 쓰는 별개 반복 작업: [[pdf-schedule-form-workflow]]

입력: 원본 PDF + 검은 펜으로 정답을 칠한 PDF 2개 → 칠을 지우고 **글자 크기에 딱 맞는
둥근 모서리 빈칸 박스**(흰 배경 + 연한 청회색 0.8pt 테두리)로 치환한 인쇄용 PDF 생성.

## 사용자 주의사항 (위반 시 재작업 지시받음 — 전부 실제 지적 이력)

1. **매우 중요한 작업 — 실수·빈칸 누락 절대 금지**. 착수 전 가능 여부 확인부터
2. **칠 얼룩 모양대로 뭉뚱그리지 말 것** — 원본 텍스트 레이어로 "가려진 글자"를 문맥 파악해
   글자 크기만큼만. 원본 PDF는 빈칸 밑 내용 참고용으로 제공되는 것
3. **다른 글자·괘선·영역 절대 침범 금지** — 옆 글자 자음 일부를 살짝 가리는 것도 안 됨
4. **여러 줄에 걸친 빈칸은 줄마다 각각 박스** — 하나로 묶으면 안 됨(두 줄인 게 티 나게)
5. 진행 전: 이해 확인 → 진행 방법 → 검증·테스트 계획 보고 → 승인 후 진행
6. 작업 폴더: `<프로젝트 루트>/pdf변환/` (참고 이미지도 여기에 올려줌), 기존 파일 덮어쓰기 금지

## 기술 핵심 (재작성 시 이 사양대로 — PyMuPDF+scipy venv 필요, 시스템 python에 없음)

- 칠 검출: 원본↔칠본 150dpi 픽셀 diff `(orig-blank>50)&(blank<210)` + closing + fill_holes
  → 칠 방식 무관 누락 불가. 칠 밑 정답 글자의 검은 픽셀은 '구멍'이라 fill_holes로 칠 영역에 포함
- **글자(char) 단위 판정 필수**: rawdict로 글자별 bbox → 획 덮임 ≥0.5 = 정답 (단어 단위는
  한글 무공백 뭉침 때문에 실패했음). 샌드위치 구제(양옆 정답 & ≥0.25), 같은 줄만 병합(줄 다르면 분리)
- 칠 제거: `true_ink(양쪽 다 어두움)` 보존 white-out 런 + 내부 침식 오버랩(이음새 방지).
  빈칸판 전용 낙서·형광펜 자국도 diff로 함께 제거
- 검증 6종 + 육안: A칠잔여0 / B박스내 비정답잉크0 / C박스높이≤줄높이 / D정답비침0 /
  E칠↔정답 매칭 전수 / F인쇄잉크 지워짐0 / G 8쪽 렌더 + 지적 지점 재현 크롭

## 작업 상태 (2026-07-13 사용자 종결)

첫 건(서양윤리 서브) v2 산출물로 **사용자 수용 완료** — "미세 글자 침범"(박스 테두리가
옆 글자 획·AA에 닿는 케이스, p1글자침범.png·p3글자침범.png)은 사용자가 알고도 넘어감.
**다음 요청부터는 이 한계를 처음부터 개선해서 시작할 것**: ① 침범 검사 잉크 임계를
<160 → <200(AA 포함)으로 강화 ② 최종 렌더에서 "박스 테두리 픽셀 ∩ 원본 글자 픽셀 = 0"
렌더 기반 검증 추가 ③ 스침 칠로 hidden 오판정될 수 있는 경계 글자(덮임 0.5~0.7)는 크롭 재검.
