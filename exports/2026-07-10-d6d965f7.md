## 실제 동작 (코드로 확인한 내용)

**저장 위치는 애초에 한 곳뿐입니다.**
`~/.claude/projects/.../memory`는 실제 폴더가 아니라 레포의 `memory/` 폴더를 가리키는 **symlink**입니다 (2026-06-11 생성 확인). 즉 memory 파일은 쓰는 순간 무조건 레포 안에 들어가고, "레포 밖 로컬 전용 저장소"라는 경로 자체가 없습니다.

**Y/N 프롬프트의 실제 의미는 "저장 허용 vs 저장 안 함"입니다.**
- **Y (Write/Edit 허용)** → 파일이 레포 `memory/`에 쓰이고, `memory-sync.js` PostToolUse 훅이 즉시 `[memory] sync: <파일명>`으로 **현재 체크아웃된 브랜치**(지금은 main)에 자동 커밋합니다 (`memory-sync.js:37`). push는 하지 않고 로컬 커밋까지만 — push는 사용자가 직접.
- **N (거부)** → Write 자체가 실행되지 않으므로 **아무 데도 저장되지 않습니다.** 로컬에 남는 것도 없습니다.

**보조 장치 2개:**
- `memory-stop-guard.js` (Stop 훅) — 세션 종료 시 미커밋 memory 변경이 남아 있으면 `[memory] auto-sync on stop`으로 커밋. 즉 어떤 경로로든 memory 파일이 변경됐다면 커밋을 피할 수 없습니다.
- `memory-pull.js` (SessionStart 훅) — 세션 시작 시 원격 최신 memory를 pull.

## 정리

| 사용자 이해 | 실제 |
|---|---|
| Y → 프로젝트 브랜치에 저장 | ✅ 맞음 (단, "프로젝트 브랜치"가 아니라 **현재 브랜치**에 로컬 커밋, push는 수동) |
| N → 로컬 PC에 저장 | ❌ 틀림 — N이면 **저장 자체가 안 됨** |

"모든 내용 요약해서 강제 저장" 같은 동작도 없습니다. 훅은 요약을 만들지 않고, 이미 Write/Edit된 memory 파일을 git commit만 합니다. 저장할 내용(개별 fact 파일 + MEMORY.md 인덱스 한 줄)을 만드는 건 훅이 아니라 Claude 본체이고, 그 시점에 permission 프롬프트가 뜨는 겁니다.
