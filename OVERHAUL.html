01 스킬 정리 — 필수만 남기기 대기

Claude 에이전트가 built-in으로 보유한 스킬과 중복되는 것들 제거. 컨벤션·프로젝트 고유 스킬만 유지.

논의 필요: humanities/education/research 스킬들은 논문 작업용 — 현재는 불필요. dream-app 스킬도 별도 앱 레포로 이관 검토.
02 훅 / 룰 정리 — 불필요 제거 대기

현재 훅 21종·룰 11종을 감사. 중복·미사용·효과 없는 것들 제거하고 핵심만 유지.

방향: 이 레포 자체에 필요한 훅 vs export 시에만 필요한 훅 구분. memory-sync 3종은 유지.
03 Export 구조화 — 템플릿별 세밀한 설정 대기

project-install.sh 8개 템플릿을 개선. 각 템플릿별 훅·룰·CLAUDE.md·에이전트·스킬을 명확하게 정의.

현황: java.md·rust.md 룰이 해당 템플릿에만 나가야 하는데 현재는 전부 포함 여부 불명확.
04 README 개편 — 간결 + 링크 구조 대기

README를 "빠른 이식 도구" 소개 문서로 재편. 상세 자산 목록은 별도 페이지 링크로 분리.

목표: README는 15초 안에 "이게 뭔지, 어떻게 쓰는지" 파악 가능한 수준으로.
05 강제성 강화 — 권고 → 훅 기반 강제 대기

현재 "권고" 수준인 규칙들을 훅으로 강제. 위반 시 경고 또는 차단.

방향: 경고(stdout 출력)와 차단(exit 2) 구분 필요. 차단은 복구 불가 위험이 있을 때만. 일반 규칙 위반은 경고.
