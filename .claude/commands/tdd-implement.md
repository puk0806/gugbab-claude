$ARGUMENTS 기능을 TDD(Red-Green-Refactor) 사이클로 구현해.

**Red 단계** — 실패하는 테스트 먼저 작성
- 구현하려는 동작을 명세하는 테스트 작성
- 테스트 실행 → 반드시 실패 확인 후 다음 단계

**Green 단계** — 테스트를 통과하는 최소 코드 작성
- 테스트가 통과할 만큼만 구현 (오버엔지니어링 금지)
- 테스트 실행 → 통과 확인 후 다음 단계

**Refactor 단계** — 코드 정리
- 중복 제거, 네이밍 개선, 구조 정리
- 테스트가 여전히 통과하는지 확인

각 단계 완료 후 다음 단계로 넘어가도 되는지 확인 요청.
$ARGUMENTS가 없으면 무엇을 구현할지 물어봐.
